# Ciara Assistant — AI Context

## What This Project Is

A personal business management app for a self-published author/freelance artist.
Local-first, privacy-first, single-user. Runs as a local web server accessed via browser.

Owner is familiar with databases (SQL) but less familiar with modern JS/TS tooling.
Explain new technologies as they come up.

## Tech Stack Summary

| Layer | Tool |
|-------|------|
| Backend | Node.js 22 + Fastify + TypeScript |
| Database | SQLite via better-sqlite3 |
| ORM | Drizzle ORM |
| Validation | Zod (shared between frontend and backend) |
| Frontend | React 19 + Vite + TypeScript |
| UI | shadcn/ui + Tailwind CSS |
| State | Zustand + TanStack Query v5 |
| Monorepo | Turborepo + pnpm workspaces |
| Auth | JWT (memory) + httpOnly refresh cookie |
| Secrets | Custom AES-256-GCM vault (Argon2id key derivation) |
| TLS | mkcert (local self-signed) |

## Monorepo Structure

```
apps/api/        ← Fastify backend
apps/web/        ← React frontend
packages/shared/ ← Shared Zod schemas + TypeScript types
data/            ← GITIGNORED: ciara.db, secrets.vault, attachments/
docs/            ← Planning documents
```

## Conventions

- All primary keys are ULIDs (not auto-increment integers, not UUIDs)
- All amounts stored as REAL, always positive; `type` field determines income vs expense
- Drizzle schema files live in `apps/api/src/db/schema/`
- Migrations generated by Drizzle, never hand-edited
- Zod schemas in `packages/shared/src/schemas/` — import from there in both apps
- Secrets never logged, never sent to frontend, never in .env files
- All API routes require JWT auth middleware (except /auth/login and /auth/refresh)
